# 参考：Manus 上下文工程原则

本技能基于 Manus 的上下文工程原则，Manus 是 Meta 于 2025 年 12 月以 20 亿美元收购的 AI Agent 公司。

## Manus 六大原则

### 原则 1：围绕 KV-Cache 设计

> "KV-cache 命中率是生产级 AI Agent **唯一**最重要的指标。"

**统计数据：**
- 输入与输出 Token 比例约为 100:1
- 缓存 Token：$0.30/MTok vs 未缓存：$3/MTok
- 10倍的成本差异！

**实现：**
- 保持 Prompt 前缀稳定（单个 Token 的变化都会使缓存失效）
- 系统 Prompt 中不要包含时间戳
- 使上下文**仅追加** (APPEND-ONLY) 并具有确定性的序列化

### 原则 2：屏蔽，不要移除

不要动态移除工具（会破坏 KV-cache）。改用 Logit 屏蔽。

**最佳实践：** 使用一致的动作前缀（例如 `browser_`, `shell_`, `file_`）以便于屏蔽。

### 原则 3：文件系统作为外部存储器

> "Markdown 是我磁盘上的‘工作记忆’。"

**公式：**
```
Context Window (上下文窗口) = RAM (易失性，有限)
Filesystem (文件系统) = Disk (持久性，无限)
```

**压缩必须是可恢复的：**
- 即使 Web 内容被丢弃，也要保留 URL
- 丢弃文档内容时保留文件路径
- 永远不要丢失指向完整数据的指针

### 原则 4：通过复述操纵注意力

> "在整个任务过程中创建并更新 todo.md，将全局计划推送到模型的近期注意力范围内。"

**问题：** 经过约 50 次工具调用后，模型会忘记最初的目标（"迷失在中间"效应）。

**解决方案：** 在每次决策前重新阅读 `task_plan.md`。目标出现在注意力窗口中。

```
Start of context: [最初的目标 - 遥远，已被遗忘]
...大量的工具调用...
End of context: [最近读取的 task_plan.md - 获得注意力！]
```

### 原则 5：保留错误的内容

> "把错误的转折留在上下文中。"

**原因：**
- 带有堆栈跟踪的失败操作让模型隐式更新信念
- 减少错误重复
- 错误恢复是“真正 Agent 行为的最清晰信号之一”

### 原则 6：不要被 Few-Shot 限制

> "统一性滋生脆弱性。"

**问题：** 重复的动作-观察对会导致漂移和幻觉。

**解决方案：** 引入受控的变异：
-稍微改变措辞
- 不要盲目复制粘贴模式
- 在重复性任务上重新校准

## 3 种上下文工程策略

基于 Lance Martin 对 Manus 架构的分析。

### 策略 1：上下文缩减 (Context Reduction)

**紧凑化 (Compaction)：**
```
工具调用有两种表示形式：
├── FULL: 原始工具内容（存储在文件系统中）
└── COMPACT: 仅引用/文件路径

规则：
- 对 STALE（较旧）的工具结果应用紧凑化
- 保持 RECENT（最近）结果为 FULL（以指导下一个决策）
```

**摘要 (Summarization)：**
- 当紧凑化达到收益递减时应用
- 使用完整的工具结果生成
- 创建标准化的摘要对象

### 策略 2：上下文隔离 (多 Agent)

**架构：**
```
┌─────────────────────────────────┐
│         PLANNER AGENT           │
│  └─ 分配任务给子 Agent           │
├─────────────────────────────────┤
│       KNOWLEDGE MANAGER         │
│  └─ 审查对话                     │
│  └─ 确定文件系统存储             │
├─────────────────────────────────┤
│      EXECUTOR SUB-AGENTS        │
│  └─ 执行分配的任务               │
│  └─ 拥有自己的上下文窗口         │
└─────────────────────────────────┘
```

**关键洞察：** Manus 最初使用 `todo.md` 进行任务规划，但发现约 33% 的动作都花在更新它上面。后来转向专用的规划 Agent 调用执行子 Agent。

### 策略 3：上下文卸载 (Context Offloading)

**工具设计：**
- 总共使用少于 20 个原子函数
- 将完整结果存储在文件系统中，而不是上下文中
- 使用 `glob` 和 `grep` 进行搜索
- 渐进式披露：仅在需要时加载信息

## Agent 循环

Manus 在一个连续的 7 步循环中运行：

```
┌─────────────────────────────────────────┐
│  1. 分析上下文 (ANALYZE CONTEXT)         │
│     - 理解用户意图                       │
│     - 评估当前状态                       │
│     - 审查最近的观察结果                 │
├─────────────────────────────────────────┤
│  2. 思考 (THINK)                         │
│     - 我应该更新计划吗？                 │
│     - 下一个逻辑动作是什么？             │
│     - 有阻碍吗？                         │
├─────────────────────────────────────────┤
│  3. 选择工具 (SELECT TOOL)               │
│     - 选择一个工具                       │
│     - 确保参数可用                       │
├─────────────────────────────────────────┤
│  4. 执行动作 (EXECUTE ACTION)            │
│     - 工具在沙箱中运行                   │
├─────────────────────────────────────────┤
│  5. 接收观察 (RECEIVE OBSERVATION)       │
│     - 结果追加到上下文                   │
├─────────────────────────────────────────┤
│  6. 迭代 (ITERATE)                       │
│     - 返回步骤 1                         │
│     - 继续直到完成                       │
├─────────────────────────────────────────┤
│  7. 交付成果 (DELIVER OUTCOME)           │
│     - 发送结果给用户                     │
│     - 附上所有相关文件                   │
└─────────────────────────────────────────┘
```

## Manus 创建的文件类型

| 文件 | 用途 | 何时创建 | 何时更新 |
|------|---------|--------------|--------------|
| `task_plan.md` | 阶段追踪，进度 | 任务开始时 | 完成阶段后 |
| `findings.md` | 发现，决策 | 任何发现后 | 查看图片/PDF 后 |
| `progress.md` | 会话日志，已做事项 | 在断点处 | 会话全程 |
| 代码文件 | 实现 | 执行前 | 出错后 |

---

## 关键约束

- **单动作执行：** 每轮一次工具调用。无并行执行。
- **必须有计划：** Agent 必须始终知道：目标、当前阶段、剩余阶段
- **文件即记忆：** 上下文 = 易失性。文件系统 = 持久性。
- **永不重复失败：** 如果动作失败，下一个动作**必须**不同
- **沟通即工具：** 消息类型：`info` (进度), `ask` (阻碍), `result` (终态)

## 关键引言

> "Context window = RAM (volatile, limited). Filesystem = Disk (persistent, unlimited). Anything important gets written to disk."
> (上下文窗口 = RAM (易失性，有限)。文件系统 = Disk (持久性，无限)。任何重要的东西都要写入磁盘。)

> "if action_failed: next_action != same_action. Track what you tried. Mutate the approach."
> (如果动作失败：下一个动作 != 相同的动作。追踪你尝试过的。改变方法。)

> "Error recovery is one of the clearest signals of TRUE agentic behavior."
> (错误恢复是真正 Agent 行为的最清晰信号之一。)

> "KV-cache hit rate is the single most important metric for a production-stage AI agent."
> (KV-cache 命中率是生产级 AI Agent 唯一最重要的指标。)

> "Leave the wrong turns in the context."
> (把错误的转折留在上下文中。)
