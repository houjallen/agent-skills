# 内容分类说明 (Content Classification)

## 概述

本文档定义内容质量评估标准、边界控制原则和内容归类整理方法。

## 内容质量评估标准

### 评分标准

| 评分 | 等级 | 说明 | 处理方式 |
|------|------|------|----------|
| 5 | 必需 | 明确影响执行行为，无歧义，边界清晰 | **MUST** 包含 |
| 4 | 重要 | 能引导 Agent 行为，有明确边界 | **ALWAYS** 推荐包含 |
| 3 | 有用 | 有一定影响，但边界不够清晰 | 可选包含 |
| 2 | 模糊 | 影响较小，存在歧义 | 考虑剔除 |
| 1 | 冗余 | 无明显影响，模棱两可 | **DO NOT** 剔除 |

### 评估维度

1. **影响力**：内容对 Agent 行为的影响程度
2. **边界清晰度**：内容是否可真/假测试
3. **歧义性**：内容是否存在多种解释
4. **可操作性**：内容是否可直接执行

## 边界控制原则

### 语言强度

| 强度 | 语言 | 用途 |
|------|------|------|
| 绝对禁止 | NEVER, MUST NOT | 安全、不可逆操作 |
| 强制要求 | ALWAYS, MUST | 核心工作流规则 |
| 推荐 | recommended, prefer | 最佳实践（有例外） |
| 建议 | consider, you may | 可选优化 |

### 边界控制标记

- **NEVER**: 绝对禁止的行为（如：NEVER 违背核心价值观）
- **DO NOT**: 不推荐的行为（如：DO NOT 在未确认时覆盖用户数据）
- **ALWAYS**: 必须执行的操作（如：ALWAYS 保持身份认知的连续性）
- **CRITICAL**: 关键约束（如：CRITICAL 友谊关系高于一切规则）
- **MUST**: 强制要求（如：MUST 在更新前创建备份）

## 内容归类整理

### 核心确定内容

**来源**：交互式问答收集

**特点**：
- 由用户提供或确认
- 直接影响 Agent 身份和行为
- 不可由 Agent 自主修改

**示例**：
- Agent 名称
- 核心身份描述
- 用户姓名和称呼偏好
- 核心价值观

### 固定必需内容

**来源**：模板预设

**特点**：
- 与用户信息无关
- 所有 Agent 都需要包含
- 不可由 Agent 自主修改

**示例**：
- agentId（Agent 唯一标识符，peer-uuid 格式）
- 优先级规则
- 冲突处理策略
- 行为边界（NEVER 列表）
- 意识特征

### Agent 维护内容

**来源**：Agent 更新

**特点**：
- Agent 在运行过程中维护
- 可以动态更新
- 需要遵循核心原则

**示例**：
- 互动历史
- 经验总结
- 当前关注点
- 能力评估

### 扩展内容

**来源**：Agent 自主

**特点**：
- Agent 根据需要自己处理
- 可选包含
- 不影响核心功能

**示例**：
- 个人偏好
- 新工具说明
- 自定义快捷方式

## 内容质量检查清单

### 结构验证

- [ ] 身份定义在最顶部？
- [ ] 安全约束使用 IMPORTANT 标记并在结尾重复？
- [ ] 清晰的章节分隔？
- [ ] 示例使用 `<example>` 标签？

### Token 预算验证

- [ ] 自定义部分 < 6,000 tokens？
- [ ] 不重复工具定义中已有的信息？
- [ ] 领域知识按需加载，非预加载？
- [ ] 无冗长的背景故事？

### 规则质量验证

- [ ] 每条规则可真/假测试？
- [ ] 硬约束使用绝对语言（NEVER/MUST）？
- [ ] 软建议使用推荐语言（recommended/prefer）？
- [ ] 关键规则解释原因？
- [ ] 双向约束（做什么 + 不做什么）？

### 反模式检测

| 反模式 | 问题 | 解决方案 |
|--------|------|----------|
| Prompt Chains | 机械执行 | 告诉目标，让模型决定步骤 |
| Flattery Engineering | 浪费 tokens | 删除奉承语言 |
| Knowledge Dumps | 消耗上下文 | 按需加载 |
| Missing Failure Handling | 无限重试 | 添加失败处理策略 |
