# 说服原则 (Persuasion Principles)

## 概述 (Overview)

LLMs 对人类相同的说服原则做出回应。这些原则帮助设计更有效的 Skill，确保在压力下也能遵循关键实践。

## 七大原则 (Seven Principles)

| 原则 | 说明 | Skill 创建应用 |
|---|---|---|
| **权威** | 专业、证书或官方来源的顺从 | "您必须"、"绝不"强制遵循标准 |
| **承诺** | 与先前行动的一致性 | 明确声明技能类型和流程 |
| **稀缺性** | 时间限制或有限可用性 | 顺序依赖："验证通过后才能打包" |
| **社会认同** | 与他人所做的一致性 | "每次"、"总是"建立规范 |
| **统一性** | 共享身份，"我们感" | "我们是 EASBot 开发者" |
| **互惠** | 归还所收到好处 | 谨慎使用 |
| **喜爱** | 与喜欢的人合作 | 避免用于合规 |

## 模式应用 (Mode Application)

| 模式 | 推荐原则 | 避免原则 |
|---|---|---|
| Tool Wrapper | 权威 + 社会认同 | 互惠 |
| Generator | 权威 + 承诺 | 喜爱 |
| Reviewer | 权威 + 承诺 + 社会认同 | 喜爱 |
| Inversion | 承诺 + 统一性 | 权威过度 |
| Pipeline | 权威 + 承诺 | 喜爱 |

## 使用示例 (Usage Examples)

**权威（强制遵循）**：
```
✅ 没有 YAML frontmatter 验证 = 不兼容。每次都需要。
❌ 有些开发者发现 frontmatter 对技能有用。
```

**承诺（明确选择）**：
```
✅ 开始创建技能前，必须明确技能类型。
❌ 考虑确定技能类型。
```
